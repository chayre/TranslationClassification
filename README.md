# CDI Cross-Linguistic Semantic Similarity Pipeline

Computes semantic similarity between two CDI (MacArthur-Bates Communicative Development Inventory) word lists from different languages, producing ranked match tables for use in bilingual child language acquisition research.

This pipeline was built using Norwegian (WS) and British English (Oxford CDI) as the example language pair. Instructions for switching to a different language pair are in the **Changing Languages** section below.

---

## Folder Structure


## How to Run

### First time (full pipeline)

**1. Install Python dependencies**
```
pip install -r requirements.txt
```

**2. Pull unilemma translation pairs from Wordbank (requires R)**
```r
install.packages("remotes")
remotes::install_github("langcog/wordbankr")
install.packages(c("dplyr", "readr"))
```
Then open `pull_unilemmas.R` in RStudio and run it. This produces `data/unilemma_pairs.csv`. Only needs to be done once unless you change languages.

**3. Run the pipeline**
```
python pipeline.py --api-key sk-ant-...
```
Or set your key as an environment variable first:
```
set ANTHROPIC_API_KEY=sk-ant-...     # Windows
export ANTHROPIC_API_KEY=sk-ant-...  # Mac/Linux
python pipeline.py
```

### Subsequent runs

If `semantic_distances.csv` already exists, skip the slow LaBSE embedding step:
```
python pipeline.py --skip-embedding --api-key sk-ant-...
```

To skip the LLM false-cognate check entirely (faster, less accurate):
```
python pipeline.py --skip-embedding --no-llm
```

## Pipeline Overview

```
CDI CSVs  ──►  LaBSE embeddings  ──►  cosine similarity matrix
                                              │
                              unilemma_pairs.csv (Wordbank)
                                              │
                                     flag verified pairs
                                              │
                                    LLM false-cognate check
                                    (batched API calls)
                                              │
                                     score each pair:
                                       unilemma     → 1.0
                                       false cognate → 0.0
                                       else          → cosine_sim
                                              │
                                    top-10 Excel output
```

**Step 1 — LaBSE embedding:** Every word from both CDI lists is encoded into a shared multilingual semantic vector space using LaBSE (Language-agnostic BERT Sentence Embeddings). LaBSE supports 109 languages and places translation equivalents close together regardless of spelling. Cosine similarity is computed for all word pairs across the two languages.

**Step 2 — Unilemma flagging:** Wordbank provides human-verified cross-linguistic translation equivalents called unilemmas. Any pair sharing a unilemma is flagged `is_unilemma_match=True` and assigned score=1.0, overriding LaBSE. This is the gold standard signal.

**Step 3 — False cognate detection:** Non-unilemma pairs where the two words are orthographically similar (Levenshtein similarity > 0.6) are sent to an LLM for false cognate checking. Words that look identical or similar but mean different things are assigned score=0.0.

**Step 4 — Output:** Top-10 matches are written to colour-coded Excel files in both directions. A second sheet in each file lists all flagged false cognates.

## Changing Languages or Datasets

To use a different language pair, four things need to change:

**1. CDI data files**

Go to [wordbank.stanford.edu](https://wordbank.stanford.edu) → Data → Download item data. Select your languages and forms (WS = Words & Sentences, WG = Words & Gestures) and download the CSVs into `data/`. Then update the file paths at the top of `pipeline.py`:

```python
NORWEGIAN_CSV = os.path.join(DATA_DIR, "norwegian_itemdata.csv")   # ← change
ENGLISH_CSV   = os.path.join(DATA_DIR, "british_english_itemdata.csv")  # ← change
```

**2. R script — language names**

In `pull_unilemmas.R`, update the language and form strings. Run `get_instruments()` in R first to see all available options, then match the names exactly as they appear:

```r
no_items <- get_item_data("Norwegian", "WS")                  # ← change
en_items <- get_item_data("English (British)", "Oxford CDI")  # ← change
```

**3. False cognate prompt**

The LLM prompt in `pipeline.py` inside `check_false_cognates()` contains a hardcoded list of known Norwegian–English false cognates. Replace this list with examples relevant to your new language pair. Also update the list of known true matches. The more accurate examples you provide, the better the detection will perform.

**4. Instrument compatibility**

Try to use the same form (WS or WG) for both languages. Mixing instruments targeting different age ranges (WG covers 8–18 months, WS covers 16–36 months) reduces unilemma overlap since the word lists target different developmental stages.

LaBSE itself requires no changes — it handles any language pair automatically.

## Output Files

### `data/semantic_distances.csv`
All word pairs with columns: `norwegian_word`, `english_word`, `norwegian_category`, `english_category`, `cosine_sim`, `is_unilemma_match`, `uni_lemma`.

### `results/top10_english_for_norwegian.xlsx` and `results/top10_norwegian_for_english.xlsx`
Top-10 semantic matches in each direction, ranked by score. Both files have two sheets:
- **Top matches** — colour-coded by score (green ≥0.75, yellow 0.60–0.75, red <0.60)
- **False Cognates** — pairs penalised for being orthographic false friends
